# PR 改动总结：窗口管理优化 + Agent 菜单功能增强

## 📋 改动概述

1. **窗口管理优化**：实现统一的窗口管理机制，防止多次点击设置链接时重复打开窗口
2. **Agent 菜单增强**：完善 Agent 总开关、键鼠控制、MCP工具三个开关的交互逻辑和视觉反馈

## ✅ 完成的任务

### 1. 窗口管理优化

**创建全局窗口管理机制**
- 新增 `window._openSettingsWindows` 跟踪已打开窗口
- 新增 `window.openSettingsWindow()` 统一管理窗口打开
- 支持自定义窗口特性、位置、大小和关闭回调
- 自动清理已关闭窗口的引用

**修复重复初始化问题**
- `setupFloatingButtons()` 添加重复初始化检查
- `createPopup()` 添加重复创建检查

**统一设置链接的窗口打开方式**
- 设置弹出框链接（记忆管理、角色设置、Live2D管理、API设置）
- Agent 弹出框菜单项（Live2D设置、API密钥、角色管理、声音克隆、记忆浏览）
- 角色管理页面的 Live2D 模型管理链接

### 2. Agent 菜单功能增强

**UI 改进**
- 添加对勾图标（✓）显示开关激活状态
- 优化禁用状态的视觉反馈（透明度、光标样式）
- 添加鼠标悬停提示（显示"不可用"状态）

**交互逻辑优化**
- 子开关（键鼠控制、MCP工具）在总开关关闭时自动禁用
- 总开关关闭时自动取消选中并禁用子开关
- 子开关只能在总开关开启时操作

**可用性检查机制**
- 总开关开启时检查 Agent 服务器状态
- 实时检查键鼠控制和 MCP 工具的可用性（每秒检查一次）
- 根据可用性动态更新开关的禁用状态和提示信息

**状态管理**
- 总开关开启/关闭时同步更新后端标志位
- 总开关关闭时自动停止所有任务并重置状态
- 子开关状态变化时同步更新后端标志位

## 🔍 影响分析

### 不会影响的功能
- ✅ 其他按钮功能（麦克风、屏幕分享等）不受影响
- ✅ 后端 API 接口保持不变，只是前端调用方式优化
- ✅ 其他页面的功能不受影响

### 需要注意的点
- ⚠️ Agent 弹出框打开时会每秒检查一次可用性，关闭时自动清理定时器
- ⚠️ 总开关关闭时会调用 `/api/agent/admin/control` 停止所有任务
- ⚠️ 所有 Agent 相关状态变更都会同步到后端 `/api/agent/flags` 接口
